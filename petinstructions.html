  • Sleepy: I don't like to play when I'm sleepy. Don't date to mess with me when I'm sleepy. I need rest too! However, study the state table to discover the thing that can make me happy when I'm sleepy!

Only For Programmers!

As shown by the state table above, Mario can have FIVE STATES and FOUR STIMULI to change those states. However, in case of INACTIVITY for 7 seconds, the state is changed AUTOMATICALLY. Also, Mario being a Dragon, it has an ability to KILL THE PERSON in EXTREME CASES (MORE Angry or MORE Hungry). In that case, page freezes and Mario can only be accessed by RELOADING the page.

Further, moving towards the code, it is clearly divided into different functions. There are a different function for changing the states (ChangeStates()), Inactivity, Beginning the Timeout, Stopping the Timeout, Extreme cases (More Hungry or Angry), and all the Stimuli. Therefore, the data represented in the table mentioned above is executed using functions.

There are TWO global variables: state (counter for changing the image with change in states) and MyVar(which is used to call and terminate the timeout). As the user chooses any button for executing an action (stimuli), it calls the respective function, and image is changed according to the data represented in the state table. However, in case of an inactivity upto 7 seconds, the state changes automatically according to the data represented in the state table above. Math.random() method has been used to change few of the states in case of inactivity. As it returns a number between 0 and 1, state changes on the basis of the generated value compared to 0.5 (i.e. a particular state is obtained if a value is greater than 0.5 and another state is obtained if value is less than 0.5). This is done to maintain equal (50/50) randomness between the states. setTimeout() and clearTimeout() methods are use for that purpose. Remember, the code gets executed only when user loads the webpage and clicks any of the button. Until user clicks any of the button, inactivity isn't considered and Mario remains in the relaxed state.

Another challenging and interesting fact about Mario lies in the "MORE Angry" and "MORE Hungry" states. The users have to be alert while dealing with Mario because once it reaches one of those states, it means that the user has reached extremes of carelessness and Mario will KILL THE USER. Hence, the user has to always be careful that Mario doesn't reach any of its EXTREME STATES. If it does, DEATH NOTICE and IMAGE appears on the screen and the webpage freezes. It has to be RELOADED to access MARIO again.
